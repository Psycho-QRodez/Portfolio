Full-stack Web Application · 2026.06

Job Application Tracker 全栈求职管理系统

在真实求职过程中，职位信息往往分散在招聘网站、邮件和个人笔记中， 导致申请进度难以追踪、状态更新容易遗漏。 为了解决这一问题，我设计并开发了 Job Application Tracker， 将职位投递、面试安排与结果管理集中到统一的平台中。

  • Pain Point： 求职信息分散，难以追踪不同岗位的最新状态。
  • Solution： 设计统一的数据模型，将职位生命周期抽象为 Applied、Interview、Offer、Rejected 四种状态， 并支持搜索与筛选。
  • Implementation： 使用 Next.js、TypeScript、Supabase 与 PostgreSQL 实现完整 CRUD、云端数据同步与自动部署。
  • Challenge： 解决前端状态更新与数据库同步一致性问题， 保证用户操作能够即时反馈并持久保存。
  • Outcome： 项目完成后成功部署上线， 支持跨设备访问与实时管理求职流程。
思考: 一个看似简单的求职管理工具，真正落地时需要同时考虑用户体验、 前端状态管理、数据库一致性以及部署维护等问题。 我意识到，工程开发不仅是功能实现，更重要的是设计清晰的数据流， 保证用户操作能够被准确、稳定地记录与反馈。

AI Chatbot Prototype · 2026.01 - 2026.04

CPL Assistant / AI Chatbot 系统开发

在课程豁免审核流程中，学生需要提交大量证明材料， 审核人员则需要反复核对文件与信息， 整体流程存在沟通成本高、信息追踪困难的问题。

  • Pain Point： 传统邮件沟通效率低， 用户难以确认材料提交进度与审核状态。
  • Solution： 将审核流程重构为聊天式交互体验， 利用 AI 辅助信息整理与摘要生成。
  • Implementation： 负责 React 聊天界面开发、 Prompt Engineering、 前后端接口联调与功能测试。
  • Challenge： 调整 Prompt 以提升回复稳定性， 平衡准确性与用户体验。
  • Outcome： 完成课程项目原型验证， 探索大语言模型在教育行政场景中的应用价值。
思考： AI 应用的价值并不完全取决于模型本身， 更取决于交互流程设计、Prompt 调优以及业务场景的理解。 一个优秀的 AI 产品，需要在准确性、易用性与用户预期之间找到平衡。
